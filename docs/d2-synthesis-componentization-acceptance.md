# D2 Synthesis 页面组件化验收清单（真实页面）

## 1. 验收目标

确认本轮 D2 改造后：

1. 页面功能不回退（合成、分段、编辑、连播、导入导出）。
2. 状态展示一致（任务、进度、stale/missing 标识）。
3. 关键交互稳定（单段重生成、已选重生成、片段编辑保存）。

## 2. 环境前置

1. 后端已启动（`uvicorn` 或项目既有启动方式）。
2. 前端已启动（`npm run dev`）。
3. 使用一个“有分段、有音频资产”的项目进行验收。
4. 同时准备一个“无任何音频资产”的项目做空态验证。

## 3. 核心流程验收（必须通过）

## 3.1 合成参数卡

1. 打开“合成导出”页，检查 `num_step / guidance_scale / gap_duration_ms / output_format / denoise` 均可修改。
2. 点击“开始合成”，任务开始，按钮进入运行态。
3. 合成中点击“停止”，状态切换为取消链路（`cancel_requested -> canceled` 或既有状态）。
4. 取消完成后，“开始合成”按钮应自动恢复可点击；不需要刷新页面。

通过标准：参数修改可生效；开始/停止可用；取消后按钮自动恢复；无 JS 报错。

## 3.2 任务状态卡

1. 合成进行时，状态、连接、进度、Task ID 有效显示。
2. 合成完成后，出现完整音频下载链接与字幕下载链接（若有）。
3. 点击“导入工程 ZIP”，能触发导入并在有 warning 时显示导入提示。

通过标准：状态信息与真实任务一致；下载链接可点击；导入入口可用。

## 3.3 完整音频卡

1. 有完整音频时显示波形播放器。
2. 可播放、可定位，时间轴与缩放交互正常（若项目启用）。
3. 若分段标注渲染失败，只显示弱提示“分段标注暂不可用，已保留完整音频播放”，且完整音频播放/缩放仍可用。

通过标准：播放器正常工作，不出现空白卡片；分段标注异常不阻断完整音频交互。

## 3.4 分段时间线

1. 有分段音频时，逐行显示：序号、角色、状态徽章、文本、播放器、动作按钮。
2. 每行“重新生成”仅触发该行重生成（不是全选重生成）。
3. “选择段落重新生成”可把推荐段落加入勾选；“重新生成已选段落”按选择执行。
4. “从此处连播”可从当前段开始连续播放；“停止连续播放”可停止。

通过标准：分段操作行为与文案一致；不会误触发整本重生成。

## 3.5 分段编辑

1. 任意一行点击“编辑”进入编辑态，字段展示完整。
2. 修改文本后“保存”，提示成功，并将该段加入待重生成集合。
3. `tts_overrides` 输入非法 JSON 时应报错并阻止保存。
4. 点击“取消”应退出编辑态且不提交改动。

通过标准：编辑闭环正确，错误提示准确。

## 3.6 空态与缺失音频

1. 选择“无任何音频资产”的项目进入页面。
2. 分段时间线不应展示可播放分段列表。
3. 需显示“缺失音频文件”空态提示。

通过标准：空态逻辑正确，不出现假数据。

## 4. 回归风险点（重点复查）

1. stale/missing 标签显示是否准确。
2. 单段重生成后该行“更新高亮”是否短暂出现并自动消退。
3. 切换项目后，编辑态、勾选态、最近更新态是否被清空。
4. 导入 ZIP 后，页面状态是否正确刷新。
5. 点击“停止”后是否会卡在“任务取消中...”且按钮无法恢复（应当不会）。
6. 完整音频波形是否出现“弱提示但可播放”的降级行为（应当可接受）。

## 5. 自动化验证（建议每次执行）

```bash
cd frontend
npm test
```

```bash

```

```bash
.\.venv\Scripts\pythonw.exe -m unittest backend.tests.test_api_smoke backend.tests.test_task_flows backend.tests.test_persistence
```

# P0 手动验收脚本：项目文件打开/保存/恢复语义

## 文档状态

- 状态：可执行验收脚本
- 范围：P0（项目文件打开复用、保存/另存语义、最近打开恢复一致性）

## 一、验收目标

确认以下语义已经成立：

1. 打开同一个 `.bvtproject.json` 时，复用同一逻辑项目，不新增副本
2. 保存项目默认回写已绑定文件
3. 另存项目会切换新的文件绑定
4. ZIP 导入与项目文件打开语义不同
5. 启动恢复后，`currentProject + script + sourceText` 同步恢复

## 二、前置条件

1. 后端和前端已启动
2. 使用同一个浏览器会话进行验收
3. 当前工作目录下有可写路径用于保存项目文件

## 三、验收步骤

### Case 1：首次保存并打开项目文件

1. 在“文本输入”创建项目 `P0-A`
2. 输入示例文本并解析（生成至少 1 个片段）
3. 点击“保存项目”，保存为 `A.bvtproject.json`
4. 记录当前项目 ID（可在 `/api/v1/projects` 中查看）
5. 点击“打开项目文件”，选择同一个 `A.bvtproject.json`

期望：

1. 页面提示“继续编辑项目”或等价语义
2. 项目列表不新增同名副本
3. 当前项目 ID 不变

### Case 2：重复打开同一项目文件

1. 再次执行“打开项目文件”并选择 `A.bvtproject.json`

期望：

1. 仍复用同一项目 ID
2. 仍不新增列表项

### Case 3：保存与另存语义

1. 修改文本或剧本内容
2. 点击“保存项目”
3. 校验 `A.bvtproject.json` 更新时间已刷新
4. 点击“另存项目”，保存为 `B.bvtproject.json`
5. 再修改内容并点击“保存项目”
6. 校验更新落在 `B.bvtproject.json`

期望：

1. 保存默认回写当前绑定文件
2. 另存后绑定切换到新路径

### Case 4：ZIP 导入语义隔离

1. 导入一个工程 ZIP
2. 观察项目列表和当前项目状态
3. 在导入项目中点击“保存项目”

期望：

1. 导入项目显示为 archive 来源
2. 导入项目不会自动绑定到某个 `.bvtproject.json`
3. 首次保存会进入选择保存位置（等价另存语义）

### Case 5：重启恢复一致性

1. 停止并重启前后端
2. 刷新页面
3. 等待启动恢复完成

期望：

1. 最近打开项目自动恢复
2. 文本输入框不为空（与项目 `script.source_text` 一致）
3. 剧本编辑页片段数量与项目实际脚本一致

## 四、建议的辅助检查接口

1. 列表：`GET /api/v1/projects`
2. 项目详情：`GET /api/v1/projects/{project_id}`
3. 剧本：`GET /api/v1/projects/{project_id}/script`

重点字段：

1. `project_origin.kind`
2. `project_origin.source_project_id`
3. `project_origin.project_file_name`
4. `project_origin.project_file_fingerprint`

## 五、通过标准

以下全部满足即判定 P0 通过：

1. 同一项目文件重复打开不新增副本项目
2. 保存/另存行为符合桌面软件直觉
3. ZIP 导入与项目文件打开语义清晰分离
4. 重启恢复后项目与脚本状态一致
